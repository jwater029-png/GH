## 一、一句话

> **星网是给"用 AI 干活的普通人"造的本地大脑——它在你和各种 AI 工具之间,记住你是谁、你怎么想、你怎么做事,让每一个 AI 都照着"你"来给你干活。用得越久,它越像你。**

---

## 二、为谁造(用户收窄)

**vibecoder ——不懂深代码,但已天天用 Claude Code / Cursor / Codex 这类 Agent 干活的人。** 创始人本人就是这群人。

这群人处境尴尬:**手里拿的是程序员的工具,脑子是普通人的脑子。** 工具全是给极客设计的(命令行、配置文件、术语),市面几十个"AI 规则管理"工具没一个为他们做,全是开发者 CLI。**这是真缝。** 创始人"不懂代码"在这里不是短板,是优势——只有这种人才会坚持把产品做成普通人能看懂、不吓人的样子。

**核心用户:** 每天 ≥ 2 小时用 Agent 工具、会在多个工具间切换、要大白话怕命令行的 vibecoder。

**不服务:** 普通好奇用户、单一工具锁定用户、不愿优化工作流的人、传统极客程序员(他们有自己的 CLI)。

---

## 三、解决什么痛(客观存在,不需要验证有没有人疼)

**表层痛:你的"自己"散落在每个工具里,而且每个都是死的。**
- 在 Claude Code 调教半天告诉它你的习惯;换 Cursor,从头再来;换 Codex,又从头再来。
- 你写死在配置文件里,过两个月自己都忘了写过啥。

**深层痛:没有任何一个地方,完整装着"你这个人怎么做事"。** AI 一个个都聪明,但都不认识你,每次都得重新认识。

> 痛点是客观的,不写"验证用户疼不疼"。我们只负责把它解决到无可替代。

## 四、产品结构:三层

```
        ┌─────────────────────────────────────────┐
        │  灵魂(脸):越用越像你的大脑              │  ← 护城河 + 产品的脸
        │  观察 → 蒸馏(半自动)→ 新陈代谢          │     就是那张节点图
        │  熄灭/点亮/锁定,遵循奥卡姆剃刀           │
        ├─────────────────────────────────────────┤
        │  主柱(身体):个人偏好的家                │  ← 价值本体
        │  个人级(跟人走)+ 跨工具 + markdown 主权 │     偏好/习惯/skill
        ├─────────────────────────────────────────┤
        │  地基(水管):跨工具同步                  │  ← 必需但不是卖点
        │  抄开源(AGENTS.md 标准),不自造          │
        └─────────────────────────────────────────┘
```

### 地基:跨工具同步(水管,抄开源)

让偏好能被 Claude Code / Cursor / Codex 都读到。**行业已有标准(`AGENTS.md`,OpenAI 发起、Linux 基金会 Agentic AI Foundation 治理),直接抄开源,不自造。** 必需的管道,但不是卖点——谁都能做。

**新增维度(2026-06-04 待排):上行吸纳。** 地基不只"下行"(星网生成配置喂各工具),还有"上行"——把各工具自带记忆(Claude auto-memory、Codex memories)读回来,汇进那一份归用户的大脑。**管道本身仍是地基(不是卖点);但"汇成一份 + 大众可掌控"的价值通过护城河②兑现。** 顺带解掉"双大脑"隐患:自带记忆 = 被吸纳的数据源,不是与星网竞争的第二大脑。需求来源见 `地基层_需求来源_创始人手搭复盘.md`(7 步手搭复盘),战略见 memory `starnet-positioning-sovereignty`。

> **⭐ 范围撑开(2026-06-06):上行吸纳不止"记忆类",还含"工程文件类"。** agent(Claude Code 等)产生/改动的工程文件也"上行"——但文件不搬(留真实项目目录),星网监听+索引它们,汇进同一份聚合视图。这条接到下面 §十二 的 `projects/` 层,详见 memory `starnet-projects-layer-scope`。

### 主柱:个人偏好的家(身体)

你的偏好、习惯、做事规矩,有一个**跟着你走、不跟项目走**的家,存在你自己电脑的 markdown 文件里。**这是你的私产,不属于任何 AI 公司**,星网哪天没了文件照样能用。解决"散落 + 死文件 + 不是我的"三个痛。

**主柱装的不只是偏好,还有工作流/skill** ——你摸索出的打法(比如"先让 AI 列大纲、我改、再写")也是"你怎么做事"的一部分,天然长在主柱上。阶段一先把偏好这条链路跑通,skill 走同一套管道,后期顺手接入(见第九节)。

### 灵魂:越用越像你的大脑(脸)= 那张节点图

星网真正值钱、别人做不了的地方。干三件事:

1. **观察** —— 接上 Agent 工具后,在旁边看你怎么干活。
2. **蒸馏(半自动)** —— 发现"你老这么干",提醒你"要不要记一条?",你点一下就存。**阶段一是半自动,不是系统自己瞎写**(理由见下"为什么蒸馏必须半自动起步")。
3. **新陈代谢(命门)** —— 节点会**熄灭和点亮**:很久没用的偏好暗下去(提醒这可能不是真需求),已内化成本能的高频习惯用另一种视觉亮起来(说明已内化)。可**锁定/钉住**重要的低频节点,永不熄灭。

**这一层的样子就是那张节点图——它不是贴在产品上的装饰皮,它就是你大脑的可视化。** 你能看见自己的思维逻辑长什么样,伸手改它,它自己迭代。用得越久,图越像真实的你。

## 五、节点图:为什么能甩开 Obsidian(命门所在)

**Obsidian 为什么中看不中用?因为它只会加,不会减。** 知识网越用越密,最后没人看得懂。

**星网的命门 = 把奥卡姆剃刀做成自动的。** 新陈代谢机制(熄灭/点亮/内化可视化)让系统自己识别哪些该暗、哪些已内化该亮,**图永远保持简洁**。没有这个机关,任何节点图迟早都变成 Obsidian。这是结构性差别,不是审美差别。

**设计红线(写进代码绕不开):**

1. **"震撼"来自留白和熄灭,不是堆视觉。** 卡冈图亚那种震撼是因为极简——一个黑洞一圈光,没别的。一想"震撼"手就容易往画面加东西,真正的纪律是克制。遵循六度分离 + 奥卡姆剃刀,不做 Obsidian 那种密密麻麻的网。
2. **不能纯按频率熄灭。** 有些偏好低频但极重要(一年用一次但绝不能错的规矩)。纯频率会误杀重要低频节点,所以必须给用户**锁定/钉住**的口子,锁住的永不熄灭。
3. **节点 = 文件。** 点节点 = 编辑那个 markdown 文件,以笔记形式记录个性化/蒸馏/日记内容。这是灵魂层的底层管道——改了图,数据要真落地。

## ⭐ 五·五、记忆分层体系(2026-06-05 定稿,核心理念)

> 创始人定调:"分层是星网的核心理念。" 设计原则:**极简、跨工具、技术小白也懂、不混乱**。关键:**按"记忆的性质"分,不按工具分**(按工具分则工具越多越乱);各工具只是出口。

**三档主轴(按"AI 该怎么对待它"分,小白一句话懂):**

| 层 | 小白话 | 进活注入? | 代谢的尺子 |
|---|---|---|---|
| 🟢 偏好层 | "AI 照着我这样干" | ✅ 是 | 看频率;亮到极致会"内化成本能" |
| 🟡 档案层 | "关于我的记录(成长/日记)" | ❌ 否 | 基本只增不减(历史不该消失) |
| 🔵 资料层 | "要用时我去查的知识" | ❌ 否 | 看常用度:常用居中、冷的靠边变暗 |

**作用域(个人/项目)只当标签,不单独成层**——否则层翻倍,违背极简。

**减法全星网统一(数据主权根基):**
- **不常用 → 变暗 + 靠边,但绝不删**(冷知识不死,只让路;一年用一次但永远对的,退到外侧暗着,随时调回)。这是对上节红线2"不能纯按频率熄灭"的统一落地。
- **唯一的删除条件 = 被推翻/不成立**(不是低频,是失效:IP 换了、方案废了)。
- **谁判推翻?甩给用户和 agent,星网不当裁判**(星网=书架只管存和摆;agent=管家干活时顺手更新/推翻)。

**星网的"活跃度"(不躺平听吩咐,也不自作主张):**
- **主动**:把垃圾/不常用/没理清的**拎出来、堆你眼前、亮给你看**,逼你正视、自己整理。
- **被动**:真正的删/改/理顺那一下,**用户拍板**,星网不替你动手。
- 比喻:不是默默收着的仓库,也不是乱扔东西的管家,是个**把你乱摊子堆到眼前、催你"该收拾了"的室友**。让星网从"仓库"升级成"逼你自我迭代的镜子"。

**核心一份、出口贴合各工具:** 核心只存一份(性质分层,工具无关),翻译只在最后送出去那刻做(adapters 出口),核心永远保持干净一份。**解掉"多元(兼容各工具)↔ 极简(小白懂)"的矛盾——核心极简,出口多元。** 呼应定位"一个大脑 + 多个 agent"(见 memory `starnet-positioning-sovereignty`)。

## ⭐ 五·六、记忆引擎升级:从"分类仓库"到"会睡觉的大脑"(2026-06-08)

> 来源:`星网_今晚讨论纪要_记忆模型与镜子定位.md`(双维度原子 + 关联网 + 镜子定位)+ 一轮对标最新脑科学与 2025-2026 记忆系统(A-MEM / Zep / Letta sleep-time / Anthropic memory tool)的核实。**结论:原三层骨架不推翻,往上叠四个原来没有的机制。** 字段落点见 `参考文档/星网_数据格式宪法.md` §5·5,**完整工程规格(公式/流程/接口/路线图)见 `参考文档/星网_记忆引擎设计.md`**,分阶段实现见执行计划。70% 的料来自纪要自己点名的缺口,30% 是用最新研究把缺口填上。

**升级一:睡眠期巩固(把蒸馏 + 代谢 + "二"合成一个引擎)。** 原来蒸馏(批次3)、新陈代谢(批次2)、主动性的"二"(远期)是三件散事。现合成一个后台"睡眠期"机制:用户空闲时回放当天事件原子 → 蒸馏出模式(生成带溯源的派生记忆)→ 同时代谢(没被强化的降权/熄灭)。**脑科学根:** 慢波睡眠里海马回放把情节记忆转成新皮层 schema,同时全脑突触缩放做遗忘(巩固 + 遗忘同时发生)。对标 Letta sleep-time compute。**守镜子哲学:睡眠期只生成觉察建议,用户确认才固化。**

**升级二:双速记忆(快慢双层),补上"工作记忆"这个最大缺口。** 原模型只有"衰减"一个标量。现按互补学习系统(CLS)分快慢:**快层 = 工作记忆 / 会话状态**(活的、当前会话、自动过期)——这是原来完全空白、却最能让 vibecoder 当场有感的点(Claude 干一半,切 Cursor/Codex 能接上下文)。**慢层 = 固化偏好 / schema**,反复强化才进核心区、几乎不衰退。快→慢的搬运 = 升级一的睡眠期。

**升级三:关联网升级成"真关系 + 时序 + 人确认"。** 原节点图按"共享标签"连边。现连真实关系(因果/演进/强化),带双时序(旧边失效 = 变暗不删,对齐"暗淡不删")、系统自动建议连接(对标 A-MEM)但连不连归用户拍(挡歪镜子)。配合检索的扩散激活(沿边召回关联),让"关联网"从存着变成用着。

**升级四:和 Claude Code 的关系讲清——互补,不是替代。** **Claude Code 记忆 = 海马(快速一次性捕捉),星网 = 新皮层 + 睡眠(巩固 / 抽象成 schema / 代谢 / 归用户)。** Claude 结构上只做了 CLS 的快系统一半;星网给它(及所有 agent)补上跨厂商、归用户、给人看的慢系统那一半。Claude 的 memory tool 本就是 client-side 文件存储、markdown 友好,星网正好垫在底下当持久层。

**战略红线(防未来自己跑偏,写死):**
1. **不在"记忆模型先进度"上跟 Mem0/Zep/Letta 卷**(那些零件已是公开货,卷不赢基础设施公司),只在 **"记忆的目的(给人看的镜子)+ 归属"** 上竞争——对手结构上做不到"跨厂商 + 归用户 + 给人看"。
2. **复杂全藏引擎、极简只留脸。** 睡眠期 / 双速 / 自动连边都是后台系统行为,**不要用户填字段**;用户永远只看到那面镜子 + 确认按钮。"顶尖的引擎 + 小白的脸"本身就是护城河,不是矛盾。

## 六、为什么蒸馏必须"半自动"起步(工程底线)

灵魂的方向是"越来越像真实的自己",这是星网为什么有人要的根本。**但有条底线死守:"灵魂指向哪"和"阶段一能交付什么"必须分开。**

**理由很硬:自动蒸馏一旦做不准,镜子就是歪的——照出一个"假的自己",比没有还糟,直接崩信任。** 这恰好踩中"解决到无可替代"原则:一面歪镜子是可替代的、不可信的。

| 层 | 是什么 | 阶段一做不做 |
|---|---|---|
| 灵魂(方向) | 数字分身,越用越像你 | 写进愿景,当北极星照路 |
| 主柱(身体) | 偏好沉淀 + 跨工具流动 | **做**,手写为主 |
| 阶段一的蒸馏 | 系统**观察→建议**,你**一键确认** | **做这个**(半自动) |
| 全自动蒸馏 | 系统自己写你的思维逻辑 | **不做**,等手写链路+频率熄灭跑通再上 |

**节点新陈代谢(熄灭/点亮/内化)阶段一就能上**——它只看客观使用数据,不需要 AI 猜你心里想啥,能落地。

## 七、两条护城河(力气全砸这)

**① 活注入** —— 按当前任务实时挑相关的几条偏好喂给 AI,不是生成一份死配置文件。
- 标准(AGENTS.md)永远只做"格式统一",不会做"按任务挑哪几条"。这是星网独占的位置。
- 三层叠加打分选 ≤5 条(全局高优先级 / 项目路径匹配 / 文件类型匹配),其余 `recall()` 按需查。

**② vibecoder 体验** —— 节点图新陈代谢 + 大白话,开发者 CLI 迈不过的坎。
- 极客做这产品,做着做着又变成 CLI;创始人"不懂代码"反成优势。
- **可见性二段(2026-06-04 深聊夯实):** 可见性分两层——**技术可见**(文件是明文,Claude auto-memory/Codex memories 也有,**不是护城河**)vs **大众可掌控**(普通人看得见/看得懂/够得着/能改,**只星网做**)。对手的记忆 = 农民签的格式合约:写在那但不知去哪看、看不懂、出事才发现"早写好了"——形式可见,实质失权。星网把可见性从 geek 隐藏目录升到普通人够得着的应用层,终极形态 = 节点图。**"浅显易懂"最难啃也最值钱,正因难,CLI 迈不过。**
- **聚合主权 = 护城河②的结构性钢筋(2026-06-04):** 为什么对手不会跨过来抢?因为"把各工具已有记忆聚合 + 归属用户 + 让普通人看懂"等于帮用户逃离自家锁定,**违背其商业利益**——这不是"谁都能做",是"**对手结构上不会做**"。对手自动记忆越强,用户大脑被切得越碎,星网聚合层越值钱:**它们不是竞品,是数据源。** 详见 memory `starnet-positioning-sovereignty`(含 1Password 类比:否定"记忆归工具",如 1Password 否定"密码归浏览器")。

**为什么是星网能做、别人做不了:**

| 谁 | 做什么 | 为什么碰不到星网的位置 |
|---|---|---|
| 标准(AGENTS.md) | 格式统一这层水管 | 永远不做"按任务挑偏好",更不做"照出你自己的镜子" |
| 几十个同类工具 | project 级 + 死文件 + 开发者 CLI | 没一个是 personal 级 + 活注入 + 给普通人的脸 |
| Obsidian | 知识网 | 只加不减,越用越密;星网有新陈代谢 |
| AgentMemory(6.8k★) | 跨工具本地记忆 | 开发者工具,目标用户跟 vibecoder 不重合 |

## 八、四件"老愿景残留"的判决(防止它们又冒出来当功能)

> 纪律:任何想法先问"它从三层的哪一层长出来"。长得出来的留,长不出或只是副产品的不立项。

| 想法 | 机制(留不留) | 卖点(能不能喊) | 阶段一立项 |
|---|---|---|---|
| 节点=文件、点节点编辑 | ✅ 留(灵魂层底层) | ✅ 用好处句式(见十一)不喊"替代文件树" | ✅ 节点图必备 |
| 防上下文漂移污染 | — | ✅ 用好处句式;❌ 不喊"防止/避免" | ❌ 活注入的副产品 |
| 省 token | — | ✅ 用好处句式;❌ 不当主卖点 | ❌ 活注入的副产品 |
| skill 成为个人资产 | ✅ 留(主柱延伸) | ✅ 能喊"你的资产换工具不丢" | ❌ 先跑通偏好再接 |

- **节点=文件:** 机制是地基必须有,但"替代文件树/解决路径堆叠"是弱痛点且在禁区,不当卖点。
- **防漂移 / 省 token:** 是活注入做对后白来的**结果**,不是功能,不立项。"防止/避免/替代"是兑现不了的承诺,违反禁区;只能用"帮/让/顺带"的好处句式说(见十一)。
- **skill:** 方向认(主柱自然延伸,amtiYo 等已在同步 skill),但阶段一不立项——偏好链路还没跑通一次,同时挖两口井两口都浅。

## 十、阶段一的真正 demo(wow 时刻)

**不是节点亮起来**(AgentMemory viewer 也做得到)。**是这个流程:**

```
1. 在星网写一条偏好:"我习惯用 pytest 写测试,函数式风格"

4.(阶段二)切到 Cursor 说"也写个测试" → Cursor 也按 pytest + 函数式写
5.(顺带)节点图里,这条偏好节点被调用时亮一下;长期高频后变成"已内化"的视觉
```

**Wow 在第 3 步——"我写的偏好真的在 AI 里活着,而且是按当前任务挑出来的那几条"。** 节点亮是顺带的好看,不是核心。

## 十一、对外说什么(好处句式 vs 承诺句式)

**铁律:说"好处"可以当卖点,说"承诺"就踩雷。** 区别在动词——`帮/让/顺带` 是好处,`防止/避免/替代/消灭` 是兑现不了的承诺,技术圈用户一眼拆穿。多承诺一分,用户多失望一分(v0.6→v2.0 大降级的教训)。

| 想表达的 | ❌ 承诺句式(踩雷) | ✅ 好处句式(能当卖点) |
|---|---|---|
| 节点=文件 | "替代你的文件树" | "点开一个节点,就是打开你大脑的一块,不用再翻文件夹找路径" |
| 漂移 | "防止上下文漂移污染" | "AI 跑偏的时候,星网帮它重新对上你是谁" |
| 省 token | "帮你省 token" | "只把当前任务相关的几条喂给 AI,它更准,顺带更省" |
| skill | (无雷) | "你摸索出的打法,沉淀成你的资产,换工具不丢" |
| 数字分身 | "你的贾维斯/数字分身" | 不主动喊,是用了多年的自然结果 |

**核心卖点(都用好处句式):**
- "你的偏好,所有 AI 都能读" / "一份本地大脑,Claude/Cursor/Codex 通用"
- "你的私有资产,不属于任何 AI 公司"
- "星网按你当前在干的活,实时挑出最相关的几条偏好喂给 AI——不是一份谁都一样的死配置"
- "用得越久,它越像你"

**禁区(任何文档/文案/对投资人都不能踩):** 见 memory `starnet-marketing-redlines`。核心:`看清 Agent 大脑在想什么 / 避免幻觉 / 防止漂移 / 替代文件树 / 成为唯一大脑 / 贾维斯`。

## 十二、数据架构(用户主权根基)

> 完整标准见 `参考文档/星网_数据格式宪法.md`,本节只给概览。

```
~/.starnet/
├── personal/           # 主柱:个人大脑(跟人走)
│   ├── preferences/    # 偏好(阶段一核心)
│   ├── habits/         # 习惯
│   ├── decisions/      # 决策风格
│   └── diary/          # 日记
├── workflows/          # 主柱:skill/工作流资产(阶段三接入)
├── projects/           # 灵魂:项目节点元数据(nodes/filters)
├── adapters/           # 地基:同步引擎生成的各家配置(claude/cursor/agents)
└── index.db            # sqlite + sqlite-vec 向量索引
```

**所有数据 markdown 可读、可迁移、星网消失也能用。这是中立和主权的根基,不打折。**

### 节点 ↔ 文件双向同步(核心交互原则)

**节点即文件,文件即节点。星网内外任何操作都互相同步。** 完整双向矩阵(新建/编辑/重命名/移动/删除/复制/目录/批量)、Git 操作、Agent 建文件、构建产物过滤、冲突保护等细节见执行计划与数据格式宪法。要点:

- **chokidar 监听**项目目录 + `~/.starnet/`,深度监听;**debounce 500ms** 合并大批变化(git pull 不疯狂闪)。
- **删除走 `~/.starnet/trash/`**(30 天自动清),不直删;冲突写 `~/.starnet/conflicts/` 可恢复。
- **新建分两种:** 项目空间任意类型落真实项目目录;个人空间限 `.md` 落 `~/.starnet/personal/`,防奇怪文件混入大脑。

### ⭐ projects 层 = 工程文件聚合呈现治理(2026-06-06 定稿方向)

**边界升维:星网从"你的 AI 大脑(偏好/记忆)"扩到"+ 工程文件 / agent 产物的聚合呈现治理层"。** 这不是新功能,是兑现上面 `projects/` 这一格——宪法早已定义它"任意类型、留真实项目目录、用 `nodes.json` 索引、`filters.json` 排除构建产物",但零实装(批次 2+ 才落)。

- **机制铁律:** 工程文件**不搬、留功能位置**(有些文件搬走就坏);星网**监听(chokidar)+索引(nodes.json)+呈现(节点图)+治理(新陈代谢)**。即"**集中呈现 + 治理,不是强行搬**"——星网帮你在一个地方看见和管全部文件,字节留在原地。
- **§8 自检:** 它长自灵魂层"节点=文件 + 新陈代谢",落在既有 projects 空间,是同一根骨头延伸,不是范围乱扩。
- 详细范围/5 条待决子问题见 memory `starnet-projects-layer-scope`。

## 十七、产品哲学(不变的原则)

1. **数据主权不退让** —— markdown + 本地,可读、可迁移、星网消失也能用。
2. **中立不退让** —— 不绑任何 Agent 厂商,星网站在中间那层,只属于用户。
3. **节点即文件不退让** —— 节点 = 文件本身,用户不学新东西、不被锁定。
4. **新陈代谢不退让** —— 图必须会减,遵循奥卡姆剃刀,绝不沦为 Obsidian。
5. **不承诺做不到的事** —— 用好处句式,不用承诺句式。能兑现才说。
6. **不要饭式讨好用户** —— 门槛用能力指标,只解决客观痛点,不张望谁鼓掌。
