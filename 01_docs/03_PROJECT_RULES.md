# GH Project Rules

## Identity

`GH` is a new product.

It is independent from Stillfield and any earlier product unless the user explicitly asks to reuse a named part.

## Hard Boundary

- Work only inside `D:\Projects\GH` for this project.
- Do not treat older projects as default references.
- Do not import old routes, UI shells, feature chains, naming, or assumptions by habit.
- Do not infer product direction from old memory topics.
- If old material is reused, it must be selected explicitly by the user.

## Allowed Carry-Over

Old material can be reused only when all three are true:

1. The user explicitly asks for reuse.
2. The reused piece is named concretely.
3. It is imported as a selected asset, not as a default template.

## Working Rule

- Start simple.
- Grow from real work, not speculative structure.
- Keep visible product state and current module state easy to read.
- Do not let tool habits silently become product decisions.
