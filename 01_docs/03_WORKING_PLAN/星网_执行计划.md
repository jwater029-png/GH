## 怎么用这份文档

- 每次开工,**先看"当前位置"**。
- 每个阶段有【门槛】,达不到不进下一阶段,回去打磨。门槛用能力指标,不用人数。
- 任务按**批次**排,批次内尽量按顺序,跨批次依赖会标注。

## 选型铁律(全局生效,2026-05-29 定)

**规矩 A — 技术方向定死,具体库到用时再定:**
- 全程纯 HTML/CSS/JS,**不引前端框架**(React/Vue 都不引)。这是"从复杂走向简单"的底线。
- 批次 2 节点图用纯 JS 图库,**候选 Cytoscape.js**,到批次 2 开工时按实时情况最终定,不锁死。

**规矩 B — 选型与借鉴不锁死,但不在执行中横跳:**
- 任何第三方库、GitHub 借鉴对象都是"当下最优解",不是终身绑定。生态变化快,锁死=主动用过时方案。
- 每个批次**开工那一刻**扫一次实时生态,找当下最强最合适的借鉴,定了就动手。
- 一个批次做到一半**不准回头重选库**;中途发现更好的 → 记进 `01_docs/04_IDEA_INBOX`,做完这批再议。
- 借鉴 = 学思路/学实现,不是无脑抄;守住星网自己的简单和本地化。

## 阶段二 · 跨工具(地基做实)— 只写目标,到了再拆

**目标:** 同一份偏好,Claude Code / Cursor / Codex 都能读。**这层是地基不是护城河——抄开源标准(AGENTS.md),不自造。**

**核心任务(粗粒度):**
- Codex/OpenCode 适配
- Cursor 适配:生成 `.cursor/rules`(注意:Cursor 是静态文件,星网只能生成/覆盖,做不到动态活注入,效果会打折——这是要量化的差异)
- 跨工具同步:抄 AGENTS.md 生态开源实现,一处改偏好处处生效
- **⭐ 上行吸纳 + 导入器(2026-06-06 排入):** 把各工具自带记忆(Claude auto-memory / Codex memories)**读回来 → 按三档分类 → 补字段 → 转格式 → 落盘**,汇成归用户的一份大脑。管道是地基(不当卖点),价值靠护城河②(聚合归属+大众可掌控)兑现;顺带解"双大脑"隐患(自带记忆=数据源不是竞品)。**范围含 agent 工程文件的索引收编,非只记忆**(文件不搬留原地、只索引呈现,接 projects 层,见 memory `starnet-projects-layer-scope`)。需求来源见 `地基层_需求来源_创始人手搭复盘.md`,战略见 memory `starnet-positioning-sovereignty`。

> **⚠️ 优先级待你拍(2026-06-04 起悬着):** 创始人核心诉求是"**任何 agent 都自动加载**"(memory `starnet-autoload-is-core`),而 Claude Code 的自动加载(hook)阶段一已做完。跨 agent 自动加载 = **adapters 引擎**,本在阶段二;但聚合主权战略说"对手结构上不会做聚合层,越早占越好"。**要不要把 adapters/上行吸纳从阶段二提前?** 你推进跨工具时定。

**【阶段二门槛(能力)】** 同一份偏好在三家都实际生效,跨工具切换零重写,各家注入精度差异可量化;自带记忆能被吸纳汇成一份。达不到不进阶段三。
