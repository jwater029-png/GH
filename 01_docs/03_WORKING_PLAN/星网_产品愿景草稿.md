## 补集元素清单

下列元素是平台（Anthropic / OpenAI / Cursor 等）结构上不会、不能、不想做的事。
未排序，未分主次。下一步要做的就是从中选出一个**第一性主张**。

### 1. 本地优先 / 数据在用户磁盘
- 部署起点：localhost 本地网页 → Electron 桌面应用
- 用户记忆物理上存在用户自己的磁盘里
- 平台一定是云端 SaaS，记忆数据存在它们服务器
- 这是数据主权的物理基础

### 2. 跨 Agent 中立
- 同时支持 Claude Code / Codex / Cursor / VS Code
- 一份记忆在多个 Agent 里通用
- Anthropic 不会做"让用户也能在 Codex 里用同一份记忆"
- OpenAI 也不会
- 只有第三方有动机做"跨 Agent 共用记忆层"

### 3. 用户对读取的主动控制权
- 用户能看见 Agent 这次读了哪些节点
- 用户能选择哪些节点这次不被读取
- 反平台黑盒：平台的 memory 是黑盒，用户不知道读了什么、记了什么、会不会用错

### 4. 寄生本地文件树
- 文件即节点，节点即文件
- 点击节点等于打开文件，编辑节点等于编辑文件
- 寄生文件树形态，不破坏原有结构
- 平台不会动用户的本地文件系统——这是 OS 层的事，超出平台边界

### 5. 双向资产（人和 Agent 都读）
- 同一份内容，用户自己也能读，Agent 也能读
- 平台的 memory 只服务 Agent，不服务人
- 用户写给自己的笔记/日记/工作流，天然是"人也能读"的资产

### 6. 视觉品牌 / 卡冈图亚美学
- 黑色深空背景，节点悬浮，引力光带连线
- 神经放电连线动效（Agent 读取节点时实时发光）
- 虫洞动效（跨宇宙连接时）
- 平台做的是工程师工具，不会做品牌化的视觉语言
- 视觉本身有传播力

### 7. 面向非技术用户
- 妈妈可以用，老板可以用，完全不懂技术的人也可以用
- 平台的 Agent 工具瞄准的是开发者和专业知识工作者
- 面向"妈妈"的 Agent 入口是平台战略上不会优先做的
