# 星网 StarNet
## 产品愿景草稿 v0.6

---

## 一句话定位

> **星网不是帮用户多找一个 AI，而是把用户稳定地接到 AI 上。**

> **它不是帮用户买更多 token，而是让每一个 token 更接近正确结果，最终让 Agent 从对话框进化成贾维斯。**

---

## 这个产品为什么存在：一个真实的故事

我在执行第一个项目的时候，文件特别乱。

项目建在 C 盘，C 盘文件繁杂，顺序混乱。我把项目移到 D 盘，然后写 `.claude` 和 `.codex` 文件让两个 Agent 有入口进入我的项目。但全局文件只能写在 C 盘，Agent 最优先读取 C 盘。于是我折中——在 C 盘的 MD 文件里加一个路标，指向 D 盘的文件，这样 Agent 才能读到。

然后打开终端，终端默认在 C 盘，我要手动切换到 D 盘。太麻烦了，我又把整个项目移回 C 盘，重新整理了一遍文件。

整理完，我发现 `.codex` 和 `.claude` 文件里还留着之前对话的旧记忆，这些旧记忆在干扰 Agent 的判断，我只能手动删掉。

最后我把全局文件和项目文件放在同一个文件夹里，加一个路标直接指向，这样不需要来回切文件，Agent 读取也不会被周围的无关文件干扰。

**我是一个懂技术的 Vibe Coder，我花了几个小时才把这些理清楚。**

普通用户遇到同样的问题，会直接放弃使用 Agent。

**我在用手工方式解决的问题，就是星网要自动解决的问题。我是星网的第一个用户。**

## 背景：我们正在经历的跃迁

乔布斯用图形界面（GUI）替代了命令行，让普通人第一次能直接操控计算机。

文件树是那个时代的产物——它是工程师给工程师设计的，是命令行的图形化形式，本质上还是工程师思维。

现在 Claude Code、Codex 做出了桌面端的 Agent 产品，但文件树没有改变。Agent 进化了，文件系统还停在原地。

**三个具体的痛点依然存在：**
- 文件夹杂乱无章，路径容易设置错误
- Agent 理解和操作时需要人工指定精确路径
- 项目进行中新旧文件混杂，Agent 读取时发生混乱，旧记忆干扰新项目

**星网要做的，就是下一代等效跃迁：成为 Agent 的大脑，以病毒的方式寄生文件树，保留文件树的全部功能，去掉它的繁杂形式，最终完成变异。**

---

## 核心理念

**星网只做一件事：帮用户把自己蒸馏给 Agent。**

不是"训练 Agent"，不是"喂数据"，是**蒸馏**。蒸馏的意思是把杂质去掉，留下本质。用户把自己最核心的思维方式、工作习惯、决策逻辑提炼出来，持续注入 Agent。Agent 理解你越深，越能按照你的方式工作，直到成为你的数字分身。

**再往里说，星网的本质，是把人编译成 Agent 可持续理解和调用的结构。**

**核心三要素：语义提示 + 视觉表达 + 逻辑网络连接**

**设计原则：奥卡姆剃刀 + 六度分离**

若无必要不增实体。简洁是地基，逻辑依然完整保留。

六度分离在星网里不是世界观装饰，而是产品结构原则：

- 不是节点越多越强，而是路径越短越强
- 节点要少，但连接要准
- 结构要轻，但可达性要高
- 星网做的不是知识堆积，而是最短路径组织

**星网不是在制造复杂网络，而是在遵循六度分离的前提下，把用户意图最快传到正确节点。**

## 如果我是用户，我直接得到什么

- 我不用每次重新解释自己是谁、项目在哪、规则是什么
- Agent 更容易拿到对的文件、对的记忆、对的上下文
- 旧记忆、脏上下文、无关信息更容易被隔离
- Agent 不再每次都像失忆的实习生，而是越来越接近持续理解我的工作对象

---

## 贾维斯问题：为什么 Agent 还是一个对话框

贾维斯了解钢铁侠的一切——他的习惯、工作方式、历史决策、正在做什么。所以贾维斯不需要每次被告知背景，直接执行。

现在的 Agent 为什么像对话框？因为它每次都不认识你。你要反复解释自己是谁、项目在哪、规则是什么。你在给 Agent 当秘书，而不是 Agent 给你当贾维斯。

**Prompt Engineering 解决的是"这次对话 Agent 理解我"，星网解决的是"永远 Agent 理解我"。**

Prompt Engineering 是一次性的人机翻译，星网是持续的人机蒸馏。

省 token 不是目的，是结果。真正的目的是让 Agent 从对话框进化成贾维斯。

星网节省的不是全部 token，而是**无效 token**：

- 少掉重复背景交代
- 少掉无关文件读取
- 少掉错误路径和错误上下文带来的试错
- 少掉反复解释用户自己的成本

所以星网不是“省 token 工具”，而是让每个 token 更接近正确结果的结构层。

---

## 防漂移：星网的底层逻辑

Agent 有上下文窗口限制，超过限制早期内容被挤出，Agent 开始忘事，判断开始漂移。

现在最有效的防漂移方法是文件化续接——把状态写进文件，下次读文件重建上下文。文件约束 Agent，防止漂移。

但这里有一个更深的问题：**文件本身也会漂移。**

C盘D盘混乱、旧记忆残留、路标失效、父子文件找不到——文件系统自己在漂移。你用文件约束 Agent，但没有东西在约束文件。

**星网是在文件之上再加一层锚点：文件约束 Agent，星网约束文件。**

```
Agent 容易漂移
    → 用文件约束 Agent
        → 文件本身也会混乱漂移
            → 用星网约束文件
```

星网通过语义节点按需激活，让 Agent 只读当前任务相关的节点，减少无效上下文、无效读取和无效推理，从而结构性地降低 token 消耗，并显著提升防漂移效果。

---

## 没有星网，这些问题能不能解决

能，但现在大多只能靠手工补丁解决。

用户可以手动整理文件、维护规则、清理旧记忆、反复补 prompt，也能勉强把问题压住。但这套方法碎、脆、重度依赖用户自己充当系统集成师。

**所以星网解决的不是“有没有办法”，而是“谁来解决”。**

星网要做的，是把这些原本分散、手工、脆弱的补丁，收敛成默认存在的产品能力。

## 病毒策略

星网像病毒一样有三个生物特性：

| 病毒特性 | 星网对应 |
|---|---|
| 寄生 | 贴在文件树上，不破坏原有结构 |
| 复制繁殖 | 节点不断生长，星网自动扩张 |
| 记忆 | 用户思维习惯和工作流持续积累 |

病毒最终目的：经过用户的同意，让本地文件树发生变异。不是强制替换，是用户用着用着产生思维依赖，主动选择让文件树进化成星网形态。用了星网之后再去用文件树，会感觉像是在用命令行。回不去了。

## 与 VS Code / Cursor / Codex 的关系

VS Code 是你工作的地方，星网是 Agent 认识你的地方。

星网解决三件他们完全没有碰的事：Agent 读了什么你不知道、你的个人记忆没有地方存、你无法把自己蒸馏给 Agent。

星网寄生在 VS Code 旁边，不竞争，只生长。

---

## 与 Obsidian 的本质区别

Obsidian 是用户喂给自己看的，星网是 Agent 喂给用户看的。

Obsidian 让人像机器一样整理知识，星网让机器像人一样理解知识。方向完全相反。

星网保留文件系统的全部逻辑能力：查询、组织、权限、版本控制。不是阉割版，是升级版。

---

## 宇宙架构：多宇宙分隔

星网不是一张图，是多个独立宇宙。防止节点无限膨胀，每个宇宙保持干净简洁。

| 宇宙 | 内容 |
|---|---|
| 工作宇宙 | 项目文件、代码、任务节点 |
| 赛博日记宇宙 | 个人思维、习惯、记忆、日记 |
| 工作流宇宙 | 自动化流程、模板 |

**跨宇宙连接机制：**

赛博日记宇宙是默认必读的，内化在 Agent 的底层，是完成任何任务的前提。Agent 每次启动，自动读取个人记忆，不需要用户手动操作。

用户看见的是一个虫洞动效——两个宇宙之间的连接通道发光激活，告诉用户"你的个人记忆已经被读取，Agent 已经认识你了"。

用户唯一能做的操作是：**选择哪些节点这次不被读取。** 其他一切自动完成。

这符合星网最核心的原则：机器做机器该做的事，用户只需要看见和确认。

## 视觉语言：卡冈图亚宇宙美学

星网的视觉风格来自星际穿越——壮观、有秩序感、在黑暗中发光。

**核心视觉元素：**

黑色深空背景，节点悬浮其中有立体感和深度感。

光晕和引力效果，节点之间的连线是有弧度的光带，模拟引力场弯曲，越相关的节点连线越粗越亮。

卡冈图亚光环，核心记忆节点周围有光环旋转，视觉上一眼分辨主次。

失重漂移感，节点缓慢漂浮，用户拖动有惯性，放手后缓慢归位。

虫洞连接动效，跨宇宙连接时出现发光虫洞入口，动画激活，告知用户两个宇宙已连接。

神经放电连线，Agent 读取节点时发光连线实时流动，像神经元放电一样传导。

**这个视觉本身就有传播力。用户第一次看见会想截图分享，不需要任何营销。**

## 产品三层架构

### 第一层：Agent 思维可视化
- Agent 调用文件时，节点从灰色变白，神经放电动画实时连线
- Agent 自动筛选项目文件，用户不需要配置路径和环境
- 用户可以在星网中调整节点，主动过滤旧记忆和无关文件
- 只激活相关节点进入上下文，token 消耗大幅降低
- 解决核心痛点：黑盒焦虑

### 第二层：赛博日记 + 个性化蒸馏
- 写日记、建节点、记录工作流，把自己持续喂给 Agent
- 用户随时看见 Agent 对自己的理解到达什么程度
- 蒸馏越深，Agent 越像自己，直到成为数字分身

### 第三层：替代文件树（终极愿景）
- 寄生 → 过滤 → 用户只活在星网里 → 文件树退为底层 → 星网成为新标准
- 用户不需要知道 C 盘 D 盘里有什么，只需要知道项目节点在星网哪里

## 节点设计：文件即节点，节点即文件

节点就是文件本身。点击节点等于打开文件，编辑节点等于编辑文件，永远同步。

非技术用户以纯文本输入，Agent 自动识别语义转换格式。技术用户可以直接操作原始格式。

> 妈妈可以用，老板可以用，完全不懂技术的人也可以用。
