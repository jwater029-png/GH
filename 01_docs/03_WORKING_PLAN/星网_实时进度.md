## 四、对外说什么(好处句式 vs 承诺句式)

**铁律:说"好处"能当卖点,说"承诺"踩雷。** 动词:`帮/让/顺带`=好处(喊),`防止/避免/替代/消灭`=承诺(禁)。

**能喊的卖点(好处句式):**
- ✅ "你的偏好,所有 AI 都能读" / "Claude/Cursor/Codex 通用"
- ✅ "你的私有资产,不属于任何 AI 公司"
- ✅ "按你当前在干的活实时挑最相关的几条喂给 AI,不是死配置"
- ✅ "用得越久,它越像你"
- ✅ "点开一个节点就是打开你大脑的一块"(不喊"替代文件树")

**禁区(承诺句式,会被拆穿):**
- ❌ "看清 Agent 大脑在想什么" / "避免幻觉" / "防止漂移" / "替代文件树" / "成为唯一大脑" / "贾维斯/数字分身"

详见 memory starnet-marketing-redlines。

## 五、技术难点(要解决到位的硬骨头)

痛点客观(跨工具重写偏好确实疼),不验证有没有人疼。要啃的硬骨头:

1. **活注入精度**——三层打分要真能按当前任务选对偏好,不塞错不漏关键的。这是护城河①的命。
2. **节点新陈代谢**——熄灭/内化要准,不误杀重要低频节点(靠锁定兜底)。这是护城河②甩开 Obsidian 的命。
3. **跨 Agent 生效程度不一**——Claude 严格、Cursor 静态文件只能生成/覆盖、Codex 又不同,精度要可量化(阶段二)。
4. **数据零损耗迁移**——markdown 本地、星网消失也能用,主权根基不打折。
5. **节点图空间感**——做出"打开自己大脑"的感觉,极简(奥卡姆+六度分离),震撼来自留白不是堆视觉。
6. **半自动蒸馏建议质量**——建议要到"你愿意点确认"的程度,不制造噪音(批次3);全自动蒸馏质量(阶段四)。
