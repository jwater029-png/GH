## 一句话:星网是什么

> **星网 = 你的 AI 记忆管家。**
>
> 你用 AI 越久,它给你攒的"你"(偏好、记录、上下文)就越多、越乱、越膨胀。星网帮你 **① 看见**这堆东西长什么样、**② 把没用的调暗沉下去**,让任何 AI 永远拿一份**干净、且归你**的"你"去干活——而不是一坨越滚越乱、还锁在人家黑盒里的东西。

## 一、给谁、解决什么痛

**给谁:** vibecoder——天天用 Claude Code / Cursor / Codex 干活、但不懂深代码、要大白话的人。**创始人本人就是,且亲手撞过这个痛**(2026-06-06 搭 Claude 环境,亲历记忆膨胀 + AI 帮倒忙,见 `地基层_需求来源_创始人手搭复盘.md`)。

**痛(客观存在,单工具就发生,不是"换工具才疼"):**
- 你用 AI 越久,它给你攒的记忆 / 工作记录就越多、越乱、越膨胀。
- 这堆东西**没人帮你整理**——旧的不会暗下去、错的还留着、重复的越叠越多。
- 而且它还散在 AI 自己的**好几套机制**里(CLAUDE.md、auto-memory、各种配置),各写各的、一团乱麻,你想理都无从下手。
- AI 每次抓一把喂给自己,抓到的常是过时的、没用的、自相矛盾的——**它拿着一坨乱东西帮你干活,反而帮倒忙。**
- 你想打开看一眼,根本看不清这里头到底装了个什么样的"你";而且这堆东西**还不归你**,在人家的黑盒里。
- **一个工具就已经这样了**——越用越疼,跟你用几个工具没关系。
- **更深一层(客观事实):你对自己"怎么想"是有盲区的。** 从来没有一个东西完整装着"你这个人怎么做事",所以你看不清自己的套路、更看不见自己思维里的问题——想进步都没面镜子可照。

## 二、产品长什么样(最终形态 = 三个部件)

```
        ┌───────────────────────────────────────────────┐
        │  脸:一面会代谢的镜子          ★ 头牌            │  ← 节点图 + "睡眠"
        │  节点图照出关系 / 状态 / 模式(你怎么想)       │     看见
        │  睡眠让没用的变暗靠边、随时点亮(不删不丢)      │     减负
        ├───────────────────────────────────────────────┤
        │  身体:一个本地大脑                  归你        │  ← 价值本体
        │  把记忆从各家 AI 的机制壳子里抽出来,            │     私产
        │  归一化成一份干净、归你的"你"                   │
        ├───────────────────────────────────────────────┤
        │  出料口:贴合同步(捎带手)                      │  ← 不再是卖点
        │  把这份干净的你,同步进各家 AI 原生要读的地方   │     顺带跨工具
        └───────────────────────────────────────────────┘
        引擎藏在底下(记忆原子 / 快慢双层 / 关联网 / 睡眠期),你看不见——
        只看到镜子和"确认"按钮。详见《记忆引擎设计》。
```

> **和三层架构的对照(同一个东西、两种切法,别晕):** 三部件(脸 / 身体 / 出料口)是给你一眼看懂的脸;三层(灵魂 / 主柱 / 地基)是内部架构叫法(见 CLAUDE.md 硬规则)。对应:**脸 = 灵魂** · **身体 = 主柱** · **出料口(贴合同步)= 从主柱单拎出来的同步出口,降为捎带手** · 还有一层 **「地基 = 跨工具同步」** 藏在最底下,必需但非卖点,归到顶层「跨所有 AI」顺带兑现。

**1. 脸 · 一面会代谢的镜子(头牌)** —— 一张节点图,加一个"睡眠"机制:
- **看见** = 节点图照出**清单照不出来的东西**:**亮度** = 哪条正热 / 哪条暗淡,**连线** = 这事因那事 / 这条推翻那条,**聚类成团** = 你的做事套路("你老这么干")。**内容那行字退到点开才显**——第一眼看到的是结构、状态、模式,不是 300 行字。(铁律:清单已经能干的,画成点再列一遍,那是装饰不是镜子。)
- **照得最深的那一层:你自己的思维习惯和盲区。** 聚类和连线照出的"你老这么干",常常是**你自己都没察觉**的套路——这正好补上人对自己思维的盲区(那个客观痛点)。镜子照的不是"你有哪些文件",是"**你这个人怎么想**"。**用得越久,它越像真实的你,你也越看得清自己。**
- **减负** = 你不用电脑时,它在后台回放、提炼你的模式、**把没用的调暗靠边**。所以它**会代谢**,不会像 Obsidian 越用越乱。
  - ⭐ **金句(比人脑强的地方):星网的代谢 = 留下人脑遗忘的好处(去噪、聚焦),扔掉坏处(真丢了调不回)。** 没用的调暗、视野只聚焦在亮的几条;但东西**没丢**,相关话题一出现、你去找,立刻重新点亮。**长得像遗忘,什么都没真丢——人脑做不到,AI 能。** 真正会"删"的只有**被你明确推翻的**(IP 换了 / 方案废了),还进回收站可恢复。
- **这是星网的核心命门**:别人都在帮你"记更多",只有它帮你"看见 + 减负"。

**2. 身体 · 一个本地大脑(归你)** —— 你的偏好、习惯、做事规矩,存在你自己电脑的 markdown 里,跟着你走、不跟项目走。**关键是那把"归你"的刀:** 现在行业的做法是把记忆**耦合进 AI 里**当留存机制(你的记忆绑它身上,你换不掉它);星网把记忆**从 AI 里抽出来、清晰交给你管**,成为归你的资产——你随时能走。**这是你的私产,星网哪天没了文件照样能用。**

**3. 出料口 · 贴合同步(捎带手,不再是卖点)** —— 各家 AI(Claude / Cursor / Codex)本来就有自己的官方读取机制,**根本不需要星网"喂"**。星网只做一件笨事:**摸清每家从哪些文件、按啥顺序读 → 把这份干净的池子,实时同步进它原生要读的那些地方(CLAUDE.md / .cursor/rules / AGENTS.md,或注册成它能来查的 MCP 源)→ 读,永远是它自己干。** 所以**没有"星网注入引擎"这回事,只有"星网同步管道"**。它顺带让任何 AI 都读到同一份你,**但这不是星网赢的地方**——星网赢在镜子和治理,不在这。
> (唯一躲不掉的笨动作:静态文件塞不下整个池子,所以"哪一坨同步进 CLAUDE.md"这个**粗选片**星网还得做;但具体"挑哪几条用"交给 AI 自己的脑子,星网不跟它比挑选。)

## 三、它和你那些 AI 的关系 + "治理"到底治什么

> **你的 AI(Claude 等)= 海马,负责当下快速记一下;星网 = 新皮层 + 睡眠,负责沉淀、抽象、变暗、归你所有。**

**"治理"不是口号,是个具体动作。** 光 Claude 一家,关于你的信息就散在 **4 套机制**里——CLAUDE.md(你写的指令)、auto-memory(它自己记的事实)、hooks(事件)、MCP(按需查),各写各的、一团乱麻,光搞懂"哪条写哪、谁读到"就耗脑子。星网做的是:

**把这团乱麻全吸进来 → 扔掉各家的"机制壳子" → 用一套干净模型重新摆(一条记忆 = 内容 × 生命状态 + 关联)→ 在星网里清楚可见、伸手可改 → 再实时同步吐回各家原生的地方。** 这就是"把 4 套机制塌缩成一张看得懂、改得动的干净图"。

**你就用一个工具,这一层就已经值了**;多用几个(Cursor / Codex),不过是同一个动作顺手多接几家。它们的自带记忆越强、机制越多,你那堆"你"被切得越碎,星网这个"吸进来、理干净、归你、再吐回去"的治理层就越值钱:**它们不是竞品,是数据源。**

## 四、护城河:为什么是星网,不是别人(力气全砸这)

**主护城河:镜子 + 减负 + 归属(= 治理层本身)。** 节点图照出你怎么想(看见)+ 睡眠代谢把没用的调暗(减负)+ 大白话 + 归用户。**这是给人看的镜子、帮人减负的管家,不是给机器记更多的仓库。**

这条壕沟为什么别人迈不过——三个理由(都是客观结构,不是安慰):

1. **不在红海。** Mem.ai / Mem0 / Zep 这些全在做同一件事:帮你**记更多、喂进去**(囤积工具)。星网是**反面**——帮你**看见 + 调暗 + 清爽**。一个往里塞,一个往外清,根本不是一条赛道。
2. **大厂结构上不会做。** 它们把记忆**耦合进自己产品**当锁定手段,商业利益是让你越攒越多、越粘越锁定。"把记忆从我这抽出来、还能搬走归你"对它们 = 帮用户逃离自家锁定 = 自残。所以"跨厂 + 归用户 + 会代谢的镜子"这件事,它们结构上做不了。
3. **越往后越值钱。** 大厂的记忆只会越加越多、机制越叠越杂,人的记忆只会越用越乱——**治理的需求随时间变大**。今天痛,明年更痛。

> **最硬的工程骨头(也正是护城河本体):** 吸进来容易(读文件),**吐回去难**——干净模型映射回各家原生槽位、和它自己写的 auto-memory 抢同一张纸,双向同步必须先定**归属**(这条归星网还是归宿主?冲突听谁的?)。这块最硬,也正是别人不做的地方。详见《工程架构》(待起草)。

> **一条战略红线:不跟 Mem0 / Zep / Letta 卷"记忆模型谁更先进"(那些是公开货),只卷"镜子 + 归属 + 会代谢"。复杂全藏引擎,极简只留脸。**

## 七、不退让的原则(就这几条)

1. **归你** —— markdown 本地,可读可迁移,星网没了也能用;记忆从 AI 里抽出来归你,随时能走。
2. **中立** —— 不绑任何 AI 厂商,星网站在中间那层,只属于用户。
3. **会代谢** —— 没用的必须变暗靠边(随时点亮、不删不丢),遵循奥卡姆剃刀,绝不沦为 Obsidian。**这是头牌,不是附属功能。**
4. **只呈现不接管** —— 镜子照给你看,你自己觉察、你拍板;AI 产出先存"待确认",不自动固化(挡歪镜子)。提醒停在"觉察级",改不改永远你决定。
5. **不承诺做不到的** —— 说好处(帮 / 让 / 顺带),不说承诺(防止 / 避免 / 替代 / 消灭)。
6. **不要饭式讨好** —— 门槛用能力指标,只解决客观痛点,不张望谁鼓掌。
7. **不卷模型先进度,只卷镜子 + 归属** —— 复杂藏引擎,极简留脸。
