# 1-① 个人偏好编辑器 — 实现方案

> 批次 1 的第一块,产品心脏的起点。目标:**让星网在你电脑上,第一次变成一个能打开、能写下一条偏好、能存成本地 markdown 的真东西。**
> 原则:从复杂走向简单、本地化、看得懂、不返工。

---

## 〇、先定的两条规矩(写进执行计划,全局生效)

**规矩 A — 技术方向定死,具体库到用时再定:**
- 全程纯 HTML/CSS/JS,不引前端框架(React/Vue 都不引)。
- 批次 2 节点图用纯 JS 图库(候选 Cytoscape.js),到批次 2 开工时按实时情况最终定,**不锁死**。

**规矩 B — 选型与借鉴不锁死,但不在执行中横跳:**
- 任何第三方库、GitHub 借鉴对象都是"当下最优解",不是终身绑定。
- 每个批次**开工那一刻**扫一次实时生态,找当下最强最合适的借鉴,定了就动手。
- 一个批次做到一半**不准回头重选库**;中途发现更好的 → 记进 `01_docs/04_IDEA_INBOX`,做完这批再议。
- 借鉴 = 学思路/学实现,不是无脑抄;守住星网自己的简单和本地化。

---

## 一、这一版做什么 / 不做什么

**做(最小能跑闭环):**
- 一个能打开的界面,左边偏好列表,右边编辑表单。
- 能新建一条偏好:填 title、正文、类型(preference/habit/decision-style)、priority、applies-to、tags。
- 点保存 → 按数据格式宪法生成合法 frontmatter + 正文的 markdown。
- 能看到已存的偏好列表,点开能改、能删。
- 严格遵守宪法:id 规则、字段顺序、updated 自动刷新、删除走 trash。

**不做(留给后面批次,现在做就是过度):**
- ❌ 跨 Agent 注入(批次 1-③)
- ❌ MCP 暴露(批次 1-②)
- ❌ 节点图(批次 2)
- ❌ 智能蒸馏建议(批次 3)
- ❌ 花哨视觉、动画、主题

---

## 二、Tauri 取舍(方案内置,不再单问)

**采用:先纯网页版,Tauri 下一步(1-① 收尾时)再套。**

理由:今天最该拿到的是"一个能跑的真东西"。先用浏览器跑通"写偏好 → 生成合法 markdown → 列表显示",存储这一版用浏览器本地存储(localStorage)+ 一键导出 .md 文件验证格式正确。等界面和逻辑跑顺,再套 Tauri 把存储换成真正写进 `~/.starnet/personal/`。

**关键:代码结构从一开始就把"存储"单独隔成一层**(一个 storage.js),网页版用 localStorage 实现,Tauri 版只换这一个文件的实现,界面和逻辑零改动。这样"先网页后桌面"不产生返工。

---

## 三、工程放哪 / 目录结构

放 `02_frontend/`(README 里这就是前端代码目录)。第一版极简:

```
02_frontend/
├── index.html          # 界面骨架
├── style.css           # 样式(简单清晰,不堆视觉)
├── app.js              # 界面逻辑:列表、表单、增删改
├── storage.js          # 存储层(网页版=localStorage;Tauri版只换这个文件)
└── starnet-format.js   # 数据格式宪法的代码实现:id生成、frontmatter拼装/解析、字段顺序
```

五个文件,职责清楚。`starnet-format.js` 是宪法的代码化,单独成文件因为后面所有批次都要复用它。

---

## 四、starnet-format.js 要落实的宪法点(写代码必查)

- id 格式 `<type-prefix>-<slug>-<short-uuid>`:pref/habit/decision 前缀;slug 从 title 生成(中文转拼音或保留,可配);4 位 uuid 系统生成。
- frontmatter 字段顺序锁死:id → type → title → created → updated → applies-to → context → priority → tags → extra → 其他。
- 默认值:applies-to 空=全适用;priority=medium;extra={}。
- 写入时自动刷新 updated;文件名 = id + .md。
- 解析时未知字段进 extra 不报错;YAML 解析失败要提示而不是崩。

---

## 五、验收标准(这一版做到这些就算过关)

1. 浏览器打开 index.html,看到偏好编辑界面。
2. 新建一条"我用 pytest、函数式风格",选 preference / priority high / tag python。
3. 保存后列表里出现这条,点开内容正确。
4. 导出 .md,内容和数据格式宪法第六节的偏好示例**字段顺序、格式完全一致**。
5. 改 title 后 updated 自动变、id 的 slug 跟着变;删除后从列表消失。
6. 全程代码你能读懂,五个文件各司其职。
