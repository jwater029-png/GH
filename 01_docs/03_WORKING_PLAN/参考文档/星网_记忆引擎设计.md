```

   用户/各 Agent ──────►│  快层 · 工作记忆(海马)                   │

   │                                                                │
   │   活跃区(高亮)   休眠区(变暗,不删)   核心区(锁定,不衰退) │
```

## 二、核心模型:记忆原子(Memory Atom)

每条记忆是一个"原子",= 既有 7 核心字段 + §5·5 追加的四组字段。这里只讲**引擎怎么用**这些字段,字段定义以宪法 §5·5 为准。

| 维度 | 字段 | 引擎拿它干什么 | 谁维护 |
|---|---|---|---|
| 内容 | `content-kind`(habit/event/knowledge/state) | **只决定走哪条代谢尺子**(衰减/内化);**注入资格不归它管,另算,见 §7·0** | 用户可选填,默认推断 |
| 生命 | `life.heat` / `life.last-used` / `life.strength` | 算亮度、决定活跃/休眠/核心区 | **系统自动(只看客观使用数据)** |
| 生命 | `life.layer`(fast/slow) | 快层=工作记忆,慢层=长期 | 系统/用户 |
| 关联 | `links[]`(kind+valid-from+invalid-at+confirmed) | 扩散激活检索、演进降权、节点图连边 | **系统建议、用户确认** |
| 溯源 | `derived-from[]` / `reason` | 派生记忆回溯、意图加权 | 睡眠期产出 / 用户可选 |

**铁律:`life.heat/strength/last-used` 永远只由客观使用数据更新,绝不让 LLM 猜"这条重不重要"**(对齐 plan 决策"新陈代谢只看客观数据")。LLM 只参与 `links` 建议和 `derived-from` 蒸馏,且产出一律 `confirmed:false` 等用户拍板。

## 三、双速记忆(CLS):工作记忆快层

> 脑科学根:互补学习系统(CLS)——快系统=海马(一次就记,易变),慢系统=新皮层(反复才记,稳定)。这是补《纪要》§2.3 自己点名的"最大缺口"。

### 3.1 快层是什么

- **载体:** `life.layer:fast` 的原子,`content-kind:state` 居多。物理上落 `~/.starnet/personal/working/`(与长期 `preferences/` 等隔离,避免污染)。
- **内容:** "当前在干什么"——当前任务、刚做的决定、临时上下文。例:"正在重构 graph.js 的连边逻辑,已砍掉共享标签方案"。
- **自动过期:** 每条带 TTL(默认会话结束或 N 小时);过期不进长期,直接归档/丢弃。**不靠用户清理。**

### 3.2 为什么它最值钱(优先做)

跨工具接上下文:在 Claude 干一半,切到 Cursor/Codex,新 agent 启动注入时带上当前工作记忆 → **不用重讲一遍**。这是 vibecoder 当场能感到"它真懂我"的点,比节点图更实用。

### 3.3 快→慢的升级(唯一通道 = 睡眠期)

快层原子**不会自己**变成长期偏好。只有睡眠期巩固发现"这个模式反复出现"(co-occurrence ↑ → strength ↑),才生成一条慢层派生记忆**建议**,用户确认才固化。这道闸防止临时上下文污染长期大脑。

## 四、关联网络:真关系 + 时序 + 人确认

> 脑科学根:schema 同化(新记忆挂到已有结构上更易巩固)、记忆是重构性的。系统对标:A-MEM(自动连接 + 新记忆触发旧记忆更新)、Zep/Graphiti(双时序边)。

### 4.1 三类边(`links[].kind`)

| kind | 含义 | 大白话 |
|---|---|---|
| `causal` | A 导致 B | "踩了 Express 的坑 → 换 Fastify" |
| `supersedes` | B 取代 A | 新方案上位,旧的降权不删 |
| `reinforces` | C、D 反复同现 | 稳定模式,互相续命 |

### 4.2 双时序(变暗不删的机制)

每条边带 `valid-from` / `invalid-at`。**演进发生时:不删旧原子/旧边,而是给旧边填 `invalid-at=今天`** → 节点图上变暗、检索时降权、但可溯源、可调回。这是"暗淡不删、只推翻才删"在边层面的落地。

### 4.3 自动建议 + 人确认(挡歪镜子)

- **建议:** 新原子写入或睡眠期跑时,系统扫历史找候选关联(向量相似 + 标签/语境共现),生成 `links` 候选,一律 `confirmed:false`。
- **确认:** 连不连、归哪类,**用户拍板**(`confirmed:true`)。未确认的边视觉上虚线、不参与强激活。
- **理由:** 一张边连错的图 = 歪镜子,直接踩红线。宁可慢,不可错。

### 4.4 扩散激活检索(让关联网从"存着"变"用着")

recall 时不止打分命中条,还沿 `confirmed:true` 的边把邻居拉进候选:
```
score(neighbor) += α · score(seed) · edgeWeight(kind)
```
一跳为主(可配两跳),`α`、`edgeWeight` 可调(causal>reinforces>supersedes)。这就是"关联续命"的检索实现。

## 五、新陈代谢:亮度、熄灭、内化、锁定

> 命门(甩开 Obsidian):图会减。脑科学根:遗忘是主动功能(突触缩放 / 主动遗忘),不是 bug。

### 5.1 亮度公式(驱动视觉 + 分区)

每条原子算一个亮度 `B ∈ [0,1]`:
```
recency = exp(-λ_layer · Δt)            # Δt = 距 last-used 的天数,λ 按档不同
B = clamp01( w_r·recency
           + w_h·norm(heat)             # 短期频率
           + w_s·norm(log(1+strength))  # 强化次数,对数避免暴涨
           + base(priority) )           # 优先级保底
if pinned(锁定): B = 1                   # 永不熄灭
```
- **活跃区** `B ≥ θ_hot`:正常高亮,进活注入候选。
- **休眠区** `B < θ_dim`:变暗 + 靠边,**不删**,相关话题出现可被扩散激活唤醒。
- **核心区** `strength ≥ θ_core`:高强度,衰减近乎关闭(`λ→0`),几乎不暗。

### 5.2 按档不同尺子(`content-kind` **只决定代谢**)

> ⚠️ **2026/06/08 修订:这张表只管"怎么代谢",不再有"进活注入?"列。** 注入资格已解耦,由 §7·0 的"普适度 × 当前相关度"打分定——别再把"档"当注入开关。

| 档(视图) | content-kind | 代谢尺子 |
|---|---|---|
| 🟢 偏好层 | habit | 看频率;heat+strength 拉满 → **内化**(换视觉,不是变暗) |
| 🟡 档案层 | event | `λ≈0`,基本只增不减(历史不该消失) |
| 🔵 资料层 | knowledge | 看常用度;冷的靠边变暗 |
| ⚡ 工作记忆 | state | TTL 过期,不走亮度 |

**为什么删那列:** 旧表把"event=❌不注入"写死,会**误杀身份恒常项**("你是创始人"是 event,可它该像偏好一样永远相关);也让 L2 有语境时**资料蹦不出来**(你正搞苹果ID,那条 knowledge 该蹦却被档挡死)。代谢和注入是两根正交轴,§7·0 解开。

### 5.3 四个动作

- **熄灭:** `B` 跌破 `θ_dim` → 进休眠区变暗。
- **点亮:** 被调用 → `heat++`、`last-used=今天`、`B` 回升。
- **内化:** 偏好层 heat 长期高 + strength 高 → 标 `internalized`,换"已成本能"的视觉(亮但不同色),不参与"提醒你记一条"。
- **锁定:** 用户钉住 → `pinned`,`B=1` 永不熄灭(兜底重要低频)。

### 5.4 删除(唯一条件)

**只有"被推翻/不成立"才删**(IP 换了、方案废了),且**判推翻 = 用户或 agent,星网不当裁判**;删除走 `~/.starnet/trash/`(30 天可恢复)。低频**永远不是**删除理由。

## 六、睡眠期巩固引擎(蒸馏 + 代谢 + "二"合一)

> 这是把《纪要》里散落三处(批次3蒸馏 / 批次2代谢 / 远期"二")合成的一个引擎。脑科学根:慢波睡眠海马回放 → 情节记忆转新皮层 schema,同时全脑突触缩放做遗忘。系统对标:Letta sleep-time compute(后台 agent 在对话间隙合并碎片、跨对话找模式、去重、归档)。

### 6.1 触发

- **空闲触发:** 用户 N 分钟无操作。
- **定时触发:** 每日固定时段(类比"夜里睡觉")。
- 只在 idle 跑,批处理,控成本(避免抢前台资源 + 省 LLM 调用)。

### 6.2 流程(伪代码)

```
function sleepCycle():
    atoms = gather(近窗口 event 原子 + fast 层原子)

    # ① 回放 + 聚类(找重复模式)——可纯算法 + 轻量 LLM
    clusters = cluster(atoms by 共现/相似/语境)

    # ② 蒸馏派生记忆(显著模式 → 候选)——用 LLM,呈现式不说教
    for c in clusters where significant(c):
        derived = distill(c)            # 例:"你的重大反复 80% 发生在深夜"
        derived.derived_from = ids(c)   # 溯源
        derived.confirmed = false       # 等用户确认
        propose(derived)                # 堆到用户眼前(镜子)

    # ③ 关联建议(新原子 vs 历史)——用 LLM,confirmed=false
    for a in new(atoms):
        for cand in findCandidateLinks(a):
            proposeLink(a, cand, confirmed=false)

    # ④ 代谢(纯算法,只看客观数据,不用 LLM)
    for a in allAtoms:
        a.life = decay(a.life)          # 更新 heat/recency
        if invalidated_by_supersedes(a): markEdge(a, invalid_at=today)
        if B(a) < θ_dim: a.zone = dormant

    # ⑤ 呈现(不自动固化)
    surface(所有 propose 的建议)          # 用户确认才落地 = 半自动
```

### 6.3 半自动闸门(守镜子哲学)

- **阶段一/批次3:** 睡眠期只**生成建议**(派生记忆 + 连边),用户一键确认才固化。系统不自己写。
- **远期(= 纪要"二"):** 全自动睡眠期固化 + 主动开口提醒。**默认关闭**,用户亲手开启;开口前用户先定义"什么值得被打扰";主动永远"建议级"不"行动级"(只呈现觉察,不替你决定)。
- **代谢例外:** ④ 代谢是纯客观数据算法,不算"AI 猜",可一直自动跑。

## 七、检索与注入(两级)

> 对齐《纪要》§2.6 + 产品决策"注入=混合分层"。

### 7·0 注入资格解耦(2026/06/08 修订,本层最要害的修正)

**病:** 旧设计注入按"档"一刀切硬开关(偏好✅ / 档案❌ / 资料❌)。两个毛病:
- **(a) 身份恒常类被误杀**——"你是创始人"落在 event 档,按规矩不注入,可它像偏好一样**永远相关**。
- **(b) L1 手里有语境却浪费**——启动时其实拿得到项目路径,UserPromptSubmit 还拿得到你刚说那句话,却**只喂给偏好打分**;资料/档案享受不到,你正搞苹果ID那条资料该蹦却被档挡死。

**解法:解耦两根正交轴。谁都不再由"档"定注入。**

| 轴 | 管什么 | 由谁定 |
|---|---|---|
| **轴① `content-kind`** | **怎么代谢**(衰减快慢 / 是否内化 / TTL) | 档(§5·2) |
| **轴② 注入资格** | **要不要喂给 AI、什么时候喂** | **不由档定 → "普适度 × 当前相关度"打分** |

- **普适度 `universality ∈ [0,1]`:** 这条"在多少语境下都该被想起"。偏好普遍高;身份恒常项(创始人/双盘环境)也高,**不管它在哪个档**;一条具体的苹果ID资料普适度低(只在搞苹果ID时相关)。可由 `always:true`(全局钉子)直接顶满,或字段标注 + 默认推断。
- **相关度 `relevance ∈ [0,1]`:** 这条和"当前语境(项目路径 + 你刚说的话)"有多贴。复用现有三层打分 + §4·4 扩散激活。

**两级注入吃这两根轴(不再问"是哪个档"):**

```
L1(无 query,会话开头):  inject = topK( score = universality )      # 普适分高的全进,不分档
L2(有 query/语境,对话中):inject = topK( score = w_u·universality + w_r·relevance(query) )
                                  + spreadingActivation(seeds, links)  # 资料该蹦就蹦
```

**一刀解耦,两个毛病同时消:** 身份恒常项靠 `universality` 高,L1 就进了(不再被 event 档挡);苹果ID资料靠 `relevance` 高,你一提它 L2 就蹦(不再被 knowledge 档挡)。

**与现有代码的接法:** `store.js` 现有 `scoreRecord` 已经是 priority(≈普适保底)+ context×4 + tags×2 + title×1(≈相关度)。**这次解耦不是推倒重写,是把"按 content-kind 一刀切的过滤"那段删掉**,改成 L1 只按普适分(priority/always)取 topK、L2 按普适+相关合分取 topK,所有档一视同仁进打分池。向后兼容:无 `universality` 字段时用 `priority` 当普适保底。

### 7.1 L1 启动注入(极简,会话开头)

```
inject = topK_universality(allAtoms, signals)   # 普适分高的全进(偏好 + 身份恒常项),不分档,top≤5
       + currentWorkingMemory()                  # 当前 fast 层工作记忆(若有)
```
立刻上手不卡顿。这是现有 `session-start-hook.js` 的扩展(打分从"偏好档过滤 + 三层"改为"全档普适打分",再加工作记忆)。**关键变化:event 档里的身份恒常项现在也能进 L1 了。**

### 7.2 L2 按需 recall(全面,对话中)

```
recall(query):
    seeds = score(query)                 # w_u·普适 + w_r·相关(三层打分+向量/BM25 RRF),全档一视同仁
    expanded = spreadingActivation(seeds, links)  # 沿确认边拉邻居(§4.4)
    return rerank(expanded)
```
检索后端:`sqlite-vec` 向量 + BM25,RRF 融合(plan §十四已定)。**全档参与**——资料/档案不再被档挡在外,凭相关度蹦出来(§7·0)。

| 阶段 | 落什么 | 字段 |
|---|---|---|
| **批次 2(优先)** | 工作记忆快层(最实用) | `life.layer:fast` + TTL |
| 批次 2 | 时序关联替换共享标签连边 | `links[]` + 双时序 |
| 批次 2 | 新陈代谢(亮度/熄灭/锁定/内化) | `life.heat/last-used/strength` |
| 批次 3 | 睡眠期半自动蒸馏建议 | `derived-from` + 待确认队列 |
| 批次 3 | 关联自动建议 + 人确认 | `links[].confirmed` |
| 远期 | 全自动睡眠期 + 主动开口("二") | 默认关闭 + 用户定义打扰 + 建议级 |

**B-分阶段原则:字段在宪法 §5·5 现在就全定义好,实现按本表节奏推,纯追加零返工。**

---

## 十、安全与主权(写死)

1. **"二"默认关闭**,用户亲手开;开口前用户先定义"什么值得被打扰";主动永远建议级、不行动级。
2. **歪镜子防线:** 所有 LLM 产出(派生记忆、连边)`confirmed:false`,用户确认才固化;质量不到不自动落地。
3. **溯源不抹:** 演进/重构只降权旧的、留 `derived-from` 链,不删原始。
4. **主权根基:** 全部 markdown 本地、可读可迁移,星网消失也能用;`index.db`/向量可重建不是本体。
5. **客观数据底线:** 代谢只看使用数据,LLM 不参与"重要性判断"。

---

## 十一、风险与缓解

| 风险 | 缓解 |
|---|---|
| 睡眠期 LLM 调用有成本 | 只在 idle/定时跑、批处理;代谢纯算法不耗 LLM |
| 蒸馏/连边质量差 = 歪镜子 | 全部 `confirmed:false` + 用户确认闸 + 溯源可回退 |
| 工作记忆污染长期大脑 | 快层物理隔离 + 唯一升级通道=睡眠期+strength 阈值 |
| 双速/睡眠期把产品做复杂 | 复杂全藏引擎(后台/系统),脸上只留镜子+确认按钮 |
| 误杀重要低频 | 锁定(`pinned`)兜底 + 暗淡不删可唤醒 |

---

## 十二、设计红线汇总(一页纸)

1. 不卷"记忆模型先进度",只卷"记忆的目的(给人看的镜子)+ 归属"。
2. 复杂藏引擎,极简留脸。
3. 代谢只看客观使用数据,不靠 AI 猜。
4. 暗淡不删,只"被推翻"才删,判推翻甩用户/agent。
5. 所有 AI 产出先 `confirmed:false`,用户拍板才固化(挡歪镜子)。
6. "二"默认关闭、用户开启、建议级。
7. 字段只能追加不能改(宪法精神)。
8. **注入资格与"档"解耦**:`content-kind` 只管代谢,注入由"普适度 × 当前相关度"打分定;身份恒常项凭普适进 L1,资料凭相关蹦 L2,谁都不被档挡(§7·0)。
9. **加档判据**:新档独立 ⟺ 它要别的档给不了的代谢尺子或注入行为;否则用标签+links(§13·2)。

## 十三、三层 vs 双维度原子(并存,不打架)

### 13·1 调和

《纪要》§2.5 说"推翻三层",老 plan §5·5 供着三层——已调和:**三层(偏好/档案/资料)= 给用户看的视图/标签;双维度原子 = 底层怎么存。** 一条原子用 `content-kind` + `type` 决定它在视图里归哪档。两套并存,见 memory `starnet-layering-system`。

### 13·2 加档判据(2026/06/08 修订,防无脑加档)

创始人质疑"三档覆盖不全"——成立。但补档不能凭感觉加,**判据钉死:**

> **一个新 `content-kind`(档)值得独立,当且仅当它需要"别的档给不了的代谢尺子,或别的档给不了的注入行为"。**
> 凑不齐这条 = 用 `type` 标签 + `links[]` 表达,**不单独成档**。

**按此判据筛(创始人质疑点名的四类):**

| 候选 | 够格独立成档? | 判据走查 |
|---|---|---|
| ⚡ 工作记忆 state | ✅ **已够格**(已是第4种 kind) | 要 TTL 过期这把别的档没有的尺子 + "仅当前会话"的注入行为 |
| 🎯 目标/计划 | 🟡 **可能够格,待定** | 要"达成即归档"的特殊代谢 + "主动提醒"的注入行为——别的档给不了。下次单独评 |
| 👥 人物关系 | ❌ 不单独成档 | 代谢/注入和现有档没差异 → `type:person` 标签 + `links` 表达足够 |
| 💡 蒸馏洞察 | ❌ 不单独成档 | 它本就是 `derived-from` 派生原子,挂 links 即可,不需要新尺子 |

**结论:现在不急着加档。** "三档"对用户是简化视图(底层早是 4 种 content-kind);真要补的是 state 已补,目标档留作下次单独拍,人物/洞察用标签+links。**这条判据是导入器分档时的准绳**——导入器照它把19条记忆分档,而不是看着像啥就新建一个档。
